🌐 **アプリのURL:** `https://gamma-gilt.vercel.app/`

## 💡 開発の背景と目的
立命館大学の学生として日々の学生生活を送る中で、既存のシステムでは「シラバスの検索が重い」「年度やセメスターの切り替えが手間」「教科書購入時の授業コードの集計が面倒」といった課題を感じていました。
これらの不便を解消し、よりスムーズに学修計画を立てられるモダンなWebアプリを自分自身の手で開発しようと思い、本プロジェクトを立ち上げました。

## ✨ 主な機能とUI/UXのこだわり

* **マルチ年度・セメスターのシームレスな切り替え**
  2026年度〜2029年度に対応。年度はドロップダウン、セメスターはタブで瞬時に切り替え可能にし、直感的な操作性を実現しました。
* **シラバス検索 & スマートフィルタ**
  選択中の学期に合わせて授業を自動抽出。フロントエンドでのフィルタリングを活用し、高速な検索体験を提供します。
* **データ保護（スナップショット機能）**
  授業追加時にシラバス情報を単なる参照ではなくコピーとして保存（スナップショット）する設計にしました。これにより、将来大学側でシラバスが更新・削除されても、ユーザーの時間割データや表示が崩れることを防ぎます。
* **クラウド同期 (Sync ID)**
  煩わしいパスワード登録を不要にし、独自の「Sync ID」を発行する仕組みを採用。複数デバイス間（スマホとPCなど）でのデータ同期を極めて低いハードルで実現しました。
* **教科書購入サポート（授業コード一括コピー）**
  登録した授業のコードをワンタップで一括集計・コピーできる機能を実装。学期始めの教科書購入時の煩わしさを解決しました。

## 🛠 技術スタック

モダンなフロントエンド環境とBaaSを組み合わせ、個人開発として運用コスト0円（Hobby/Freeプラン）でスケール可能なアーキテクチャを採用しています。

* **Framework:** Next.js 14 (App Router), React
* **Language:** TypeScript
* **Styling:** Tailwind CSS
* **Icons:** Lucide-React
* **Database & Auth:** Supabase (PostgreSQL)
* **Hosting / CI/CD:** Vercel, GitHub

## 🔧 システム構成・インフラ

* **Vercelによる自動デプロイ**: GitHubリポジトリのmainブランチへのPushを検知し、Vercelへ自動デプロイ（CI/CD）される環境を構築しています。
* **Supabaseとの連携**: バックエンド環境を構築せず、SupabaseをHeadless DBとして利用。環境変数 (`NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`) をVercel側に設定し、セキュアにデータをやり取りしています。
